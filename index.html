Godot 4 (.NET/C#) - UI Implementációs Útmutató

A HTML/CSS/JS alapú dizájnt viszonylag egyszerűen át lehet ültetni Godot-ba, mivel a Godot Control node rendszere nagyon hasonló logikával (dobozmodell, margók, konténerek) dolgozik, mint a webes CSS.

1. A Témarendszer és Színek (CSS Variables -> Godot Theme)

A weben CSS változókat (--bg-panel, --btn-main) használtunk. Godot-ban erre a Theme (Téma) erőforrás (.tres) való.

  • A FileSystem ablakban hozz létre egy új Theme erőforrást (pl. classic_rts.tres).
  • Ezt a témát rendeld hozzá a legfelső (Root) Control node-odhoz az Inspectorban a Theme tulajdonságnál.
  • A témaváltást (Classic, Modern, Tactical) 3 külön .tres fájllal és egy egyszerű C# kóddal oldhatod meg:
// Témaváltás C#-ban public void SetThemeClassic() { Theme = GD.Load<Theme>("res://themes/classic_rts.tres"); }
2. HTML elemek megfeleltetése Godot Node-okra

Elrendezés (CSS Flexbox -> Godot Containers)

  • flex-direction: row; Használj HBoxContainer-t.
  • flex-direction: column; Használj VBoxContainer-t.
  • padding: 20px; Használj MarginContainer-t a dobozok körül.
Panelek (.panel -> PanelContainer)

A kártyákhoz használj PanelContainer node-ot. A stílust a Témán belül egy StyleBoxFlat hozzáadásával add meg (Bg Color, Border, Corner Radius, Shadow).

Gombok (.btn -> Button)

Használj StyleBoxFlat-et. Normal állapot: vastagabb alsó border. Pressed állapot: vékonyabb alsó border, és a Content Margin Top növelésével imitáld a lenyomódást.

Egyéb elemek
  • Csúszka HSlider
  • Folyamatjelző ProgressBar
  • Kapcsoló/Toggle CheckButton
3. Az Oldalmenü és a Tabok Logikája
A) TabContainer (Beépített, egyszerű)

Tegyél le egy TabContainer-t, rejtsd el a felső füleket (Tabs Visible = off). Minden fül egy belső Control lesz.

private void OnDashboardBtnPressed() { GetNode<TabContainer>("MainTabContainer").CurrentTab = 0; } private void OnLobbyBtnPressed() { GetNode<TabContainer>("MainTabContainer").CurrentTab = 1; }
B) Visible property (Rugalmas, JS-szerű)

Minden "oldal" egy külön UI Jelenet (Scene) vagy konténer, amelynek a láthatóságát (visible) állítjuk.

private void OnMenuItemSelected(string page) { GetNode<Control>("DashboardPanel").Visible = (page == "dash"); GetNode<Control>("LobbyPanel").Visible = (page == "lobby"); // ... }
4. Ikonok (FontAwesome) & Komponensek

Ikonok: Töltsd le a FontAwesome .ttf fájlját, állítsd be egy Label fontjának, és másold be az ikon unicode karakterét a Cheatsheet-ről (pl. ).

Komponens alapú fejlesztés: Ahogy weben újrahasznosítottuk a .btn-t, Godot-ban mentsd el a gombot vagy a jelvényeket külön Scene-ként (Jelenetként) (pl. CustomButton.tscn). Ezeket a jeleneteket aztán úgy húzhatod be a fő felületre, mint a Lego kockákat! (Készíthetsz hozzájuk [Export] változókat C#-ban is, pl. szín beállítására).

5. UI Események és Szignálok (C#)

A JavaScript addEventListener megfelelője a Godot-ban a Szignálok (Signals) rendszere. C# esetében a legtisztább, ha ezeket kódból, események (events) segítségével kötöd be a _Ready() metódusban, ahelyett, hogy az Editort használnád.

public override void _Ready() { // Gomb lekérése és esemény feliratkozás (mint a JS onclick) Button dispatchBtn = GetNode<Button>("DispatchButton"); dispatchBtn.Pressed += OnDispatchButtonPressed; // Slider értékváltozás (mint a JS oninput) HSlider speedSlider = GetNode<HSlider>("SpeedSlider"); speedSlider.ValueChanged += OnSpeedSliderChanged; } private void OnDispatchButtonPressed() { GD.Print("Egységek riasztva!"); } private void OnSpeedSliderChanged(double value) { GetNode<Label>("SliderValueLabel").Text = $"{value}x"; }
6. CSS Tranzíciók kiváltása (Godot Tweens)

A weben CSS-t használtunk (pl. transition: background-color 0.2s vagy animation: fadeIn). Godot-ban az ilyen UI animációkhoz a beépített Tween osztály a legjobb eszköz, ami tökéletes fade-in effektekhez vagy csúszkák/progress barok sima mozgatásához.

// Fade-in animáció egy lebegő menü megnyitásakor private void ShowDropdownMenu(Control menu) { menu.Visible = true; menu.Modulate = new Color(1, 1, 1, 0); // Teljesen átlátszó kezdet Tween tween = GetTree().CreateTween(); // 0.2 másodperc alatt felhúzzuk az alpha (átlátszóság) értéket 1-re tween.TweenProperty(menu, "modulate:a", 1.0f, 0.2f) .SetTrans(Tween.TransitionType.Sine) .SetEase(Tween.EaseType.Out); } // Progress bar értékének sima, animált növelése private void AnimateProgressBar(ProgressBar bar, double targetValue) { Tween tween = GetTree().CreateTween(); tween.TweenProperty(bar, "value", targetValue, 0.5f); }
7. Egyedi Komponensek Készítése (C# Szkriptek)

A komplexebb HTML struktúrákat (mint a saját Toggle kapcsolónk vagy az ikonnal ellátott gombok) érdemes egyedi C# szkripttel ellátott jelenetekként (Scene) megírni. Az [Export] attribútummal olyan tulajdonságokat adhatsz nekik, amiket utána a Godot Editorban egy kattintással állíthatsz.

using Godot; public partial class TacticalToggle : CheckButton { // Ez megjelenik a Godot Editorban a csomagra kattintva! [Export] public string OnText { get; set; } = "ON"; [Export] public string OffText { get; set; } = "OFF"; public override void _Ready() { // Rácsatlakozunk a saját Toggled eseményünkre Toggled += OnToggleStateChanged; UpdateLabelText(ButtonPressed); // Alapállapot beállítása } private void OnToggleStateChanged(bool isPressed) { UpdateLabelText(isPressed); // Itt jöhet egy Tween animáció a kapcsoló gombjára... } private void UpdateLabelText(bool isPressed) { Text = isPressed ? OnText : OffText; } }
